This API is simplified. It dispenses with multipart mime, which is used to make multiple changes in one http transaction. The first part of the url is the action.
  • POST /permission/robf/resume.pdf - Post rego permissions here first to spell out security restriction on file
  • POST /files/robf/resume.pdf - Just post the file
  • POST /append/robf/resume.pdf - Append to the file, for things like syslog, video streaming
  • POST /eof/robf/resume.pdf - No more file appends will happen
  • POST /meta/robf/resume.pdf - Just post metadata regarding the file (separately!)
  • POST /files/robf/myApp - Post app.tar under myApp installed. This is critical for CMS functionality.
